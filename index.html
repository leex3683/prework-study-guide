HTML

  • The head element contains information about the webpage.
  • Link to the CSS file and the JS file within the HTML using:
    • <link rel="stylesheet" href="./assets/style.css">
    • <script src="./assets/script.js"></script>
  • The body element represents the visible content shown to the user.
  • Sample link code: <a href="URL">link text</a>

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • See HTML notes above for instructions on linking to HTML file

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • See HTML notes above for instructions on linking to HTML file
  • Sample Code
    • for (var x = 0; x < array.length; x++) { console.log(array[x]); }
    • function myFunction() {}
    • var array = [0, 1, 2, 3]
    • var randomVariable = randomVariable[Math.floor(Math.random() * randomVariable.length)]

Git

  • git clone git@github.com:leex3683/prework-study-guide.git Copy repo to local machine using SSH
  • git remote -v allows fetching/pushing
  • git status checks what branch we are currently on
  • git checkout [enter branch] moves branches
  • Typical workflow
    • Create new feature branch using: git checkout -b branch-name: creates a new branch and switches to it
    • Make changes to code/files where needed, then save code/files
    • Then stage with "git add -A"
    • Then commit with "git commit -m "comment about change made" "
    • may be asked for author identity... Use

      git config --global user.email "you@example.com"

      or

      git config --global user.name "Your Name"

    • git pull origin main (ensure main is up to date)
    • git push origin feature/starter-code (pushes commit to github)
